# 🤖 Generative AI Product Strategy

## 💡 Overview
Generative AI (LLMs, Diffusion Models) is shifting the paradigm from "deterministic" software to "probabilistic" experiences. As a PM, the strategy moves from building features to building **Retrieval-Augmented Generation (RAG)** systems and managing **Model Drift**.

## 🏗️ Core Principles
1. **The Trust Gap**: GenAI can hallucinate. Building a "Human-in-the-loop" or robust validation layer is critical.
2. **Context is King**: The quality of the output depends on the quality of the prompt and the data retrieved (RAG).
3. **From UI to UX**: Moving beyond "Chat" interfaces to "Invisible AI" that augments existing workflows.

## 🎯 Practical Application
- **Prompt Engineering for PMs**: Understanding how to structure prompts to get reliable product behavior.
- **Evaluation Frameworks**: Using "Golden Sets" (human-verified correct answers) to measure model performance.
- **Cost vs. Latency**: Choosing between GPT-4o (high intelligence, high cost) and smaller models like Llama 3 (lower cost, faster).
