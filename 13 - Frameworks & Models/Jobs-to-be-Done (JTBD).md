# 🛠️ Jobs-to-be-Done (JTBD)

## 📖 Definition
JTBD is a framework for understanding customer behavior. It posits that customers don't buy products; they "hire" them to do a "job" (progress they want to make in a specific circumstance).

## 🎯 When Would I Use This?
1. **Market Discovery**: When looking for new opportunities by identifying "underserved" jobs.
2. **Product Innovation**: When you want to innovate beyond existing feature sets by focusing on the underlying motivation.
3. **Marketing/Positioning**: To write copy that speaks to the "struggle" the user is facing.

## 🏗️ How to Apply It
1. **The Job Story**: Use the format: *When [Situation], I want to [Motivation], so I can [Expected Outcome].*
2. **Interviewing**: Conduct "Switch Interviews" to find out why people stopped using one product and hired another.
3. **Focus on Circumstance**: The context matters more than the demographic. (e.g., A milkshake is hired for a long, boring commute, not because the user is a 30-year-old male).

## ⚠️ Common Pitfalls
- **Ignoring Emotional Jobs**: Focusing only on the functional task and forgetting the social or emotional "job."
- **Over-complicating**: Turning a simple insight into a massive, unusable matrix.
