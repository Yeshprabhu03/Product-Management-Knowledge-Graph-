## 💎 What Makes This Different

- **AI-Native Philosophy** — Built for the modern PM, with deep dives into RAG, Fine-tuning, AI UX patterns, and Responsible AI guardrails.
- **Embedded Real-World Cases** — Every node contains a verified industry example (Netflix, Slack, Figma, OpenAI, Stripe, Airbnb, GitHub, etc.) with a specific situation, decision, and measurable outcome.
- **Dense Cross-Linking** — Over 120+ manual bi-directional links (e.g. *Model Hallucination* → *Confidence Indicators* → *User Trust Metrics*).
- **15 Color-Coded Domains** — Visually distinct branches that allow for intuitive spatial navigation in Obsidian's Graph View.

## 🛠️ Tech Stack & Local Setup

### System Components
- **Obsidian**: Local graph-based knowledge management.
- **Quartz 4**: Lightning-fast static site generation for the web experience.
- **GitHub Actions**: Automated CI/CD pipeline for building and deploying the site.

### Running Locally (Quartz)
If you want to modify the site engine or preview changes locally:
1.  **Clone the Repo**: `git clone https://github.com/Yeshprabhu03/Product-Management-Knowledge-Graph-.git`
2.  **Install Dependencies**: `npm install`
3.  **Run Dev Server**: `npx quartz build --serve`
4.  **View Site**: Open `http://localhost:8080` in your browser.

### Using in Obsidian
1.  Open Obsidian.
2.  Click "Open folder as vault".
3.  Select the root folder of this repository.
4.  **Graph View Settings**: To see the color-coded domains, set up "Groups" in the Graph View based on the folder numbers (e.g., `path: "01 - Strategy & Vision"` → Color).

## 🤝 How to Use & Navigate

**Online:** [Explore the live graph](https://Yeshprabhu03.github.io/Product-Management-Knowledge-Graph-/) — use the global search or the interactive graph in the sidebar to jump between concepts.

**Non-Linear Discovery:** This graph is designed to be explored non-linearly. Start with a concept you're working on (e.g., *Hallucination*) and follow the "Connected Concepts" links to discover how it impacts *User Trust Metrics*, *Confidence Indicators*, and *GTM Positioning*—all from a single node.

---

> [!TIP]
> Every node follows a standardized template: **Definition** → **Why It Matters** → **How to Apply** → **Real-World Example** → **Pitfalls** → **Connected Concepts**. This ensures consistent depth and practitioner-level utility across the entire graph.
