# 🛠️ Kano Model

## 📖 Definition
The Kano Model is a theory for product development and customer satisfaction that classifies customer preferences into five categories: Must-be, One-dimensional, Attractive, Indifferent, and Reverse.

## 🎯 When Would I Use This?
1. **Feature Selection**: When you need to balance "maintenance" features with "delighter" features.
2. **User Research**: After conducting surveys to see which features actually move the needle on satisfaction.
3. **Product Positioning**: To understand what makes your product uniquely attractive vs. just meeting basic expectations.

## 🏗️ How to Apply It
1. **Identify the Features**: List the features you are considering.
2. **Functional vs. Dysfunctional Questions**: Ask users how they feel if the feature is present vs. if it is absent.
3. **Categorize**:
    - **Must-be**: Basic requirements (e.g., a phone making calls).
    - **One-dimensional**: The more you provide, the better (e.g., battery life).
    - **Attractive**: Unexpected delighters (e.g., first-time Siri integration).
    - **Indifferent**: Users don't care.

## ⚠️ Common Pitfalls
- **Feature Creep**: Focusing too much on "Attractive" features while neglecting "Must-be" basics.
- **Static Thinking**: Yesterday's "Attractive" features become today's "Must-be" basics.
